# 多语言检查 - 用户界面优化说明

## 📋 优化目标

将技术性的错误信息转化为非技术人员也能轻松理解的友好提示。

---

## 🔄 改进对比

### 改进前 ❌

用户看到的是这样的技术性错误信息:

```
{severity: "error", message: "Possible spelling mistake found.", shortMessage: "Spelling mistake"}
规则: MORFOLOGIK_RULE_EN_US
类别: Possible Typo
```

**问题**:
- 不知道哪个词有问题
- 不知道应该改成什么
- 技术术语太多 (severity, rule, category)
- 信息层次不清晰

### 改进后 ✅

用户现在看到的是清晰的结构化信息:

```
┌─────────────────────────────────────┐
│ 拼写错误                             │  ← 问题类型(中文标签)
├─────────────────────────────────────┤
│ 有问题的文字:                        │
│   "exmaple"                         │  ← 高亮显示错误的词
├─────────────────────────────────────┤
│ 问题:                                │
│   Spelling mistake                  │  ← 问题描述
├─────────────────────────────────────┤
│ 建议改为:                            │
│   ✓ example                         │  ← 修改建议
├─────────────────────────────────────┤
│ 上下文:                              │
│   This is an exmaple text...        │  ← 完整句子
└─────────────────────────────────────┘
```

## 🎨 界面改进详情

### 1. 问题类型标签
- **拼写错误** (misspelling)
- **语法错误** (grammar)
- **排版问题** (typographical)
- **风格建议** (style)
- **其他问题** (uncategorized)

使用彩色标签,一目了然。

### 2. 错误文字高亮
```
有问题的文字: "exmaple"
              ^^^^^^^^
              黄色边框 + 红色高亮
```

直接告诉用户哪个词有问题。

### 3. 清晰的问题描述
```
问题: Spelling mistake
```

简洁明了的说明。

### 4. 直观的修改建议
```
建议改为:
  ✓ example    ← 绿色背景,hover 高亮
  ✓ examples
  ✓ sample
```

最多显示 3 个建议,垂直排列,易于阅读。

### 5. 上下文预览
```
上下文:
  This is an exmaple text with som spelling mistakes.
          ^^^^^^^
          在完整句子中看到错误位置
```

## 🚀 使用指南

### 查看检查结果

1. 访问: http://localhost:5173/tools/multilingual
2. 输入网页 URL 和选择语言
3. 点击"开始检查"
4. 展开语言结果查看详细错误

### 理解错误信息

每个错误卡片包含:
- **顶部标签**: 问题类型 (拼写/语法/风格等)
- **黄色框**: 有问题的具体文字
- **问题说明**: 错误的简短描述
- **绿色建议**: 推荐的修改方案 (最多 3 个)
- **底部上下文**: 完整句子预览

### 修复错误

1. 找到"有问题的文字"
2. 查看"建议改为"中的选项
3. 在原网页中找到这段文字
4. 替换为建议的正确拼写
